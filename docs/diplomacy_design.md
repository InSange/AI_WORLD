## 1. 친밀도 (Affinity) 시스템

### 기본 개념

친밀도는 두 종족 간의 관계를 수치로 표현한다.
- **범위**: -100 ~ +100
- **방향성**: A→B와 B→A는 **독립적**으로 관리 (비대칭 가능)
- **소수점**: 내부적으로는 float 사용, 표시는 정수

```
 -100    -60     -30      0      30      60     100
   │       │       │       │       │       │       │
  전쟁    적대    냉전    중립    우호    동맹    혈맹
  WAR   HOSTILE  COLD  NEUTRAL FRIEND  ALLIED  BOND
```

### 친밀도 단계 (Affinity Level)

| 단계 | 범위 | 이름(한) | 이름(영) | 주요 특징 |
|------|------|---------|---------|---------|
| 7 | +80 ~ +100 | 혈맹 | BOND | 자원 완전 공유, 공동 전쟁 의무 |
| 6 | +50 ~ +79 | 동맹 | ALLIED | 군사 협력, 교역 우대 |
| 5 | +20 ~ +49 | 우호 | FRIEND | 교역 허용, 정보 공유 |
| 4 | -10 ~ +19 | 중립 | NEUTRAL | 불간섭, 기본 교역 |
| 3 | -30 ~ -11 | 냉전 | COLD | 교역 제한, 긴장 관계 |
| 2 | -60 ~ -31 | 적대 | HOSTILE | 교역 금지, 침략 가능 |
| 1 | -100 ~ -61 | 전쟁 | WAR | 적극적 군사 충돌 |

## 2. 친밀도 변화 규칙

### 자연 감쇠 (Passive Decay)
관계는 시간이 지나면 **중립(0)** 쪽으로 서서히 수렴한다.

```
decay_rate = 0.1 per_season   # 계절당 10% 중립으로 복원
```

적대적 관계도 아무 일 없으면 시간이 지나 냉전 → 중립으로 완화된다.

### 이벤트에 의한 변화

| 이벤트 | 친밀도 변화 |
|--------|-----------|
| 교역 성공 | +2 ~ +5 |
| 교역 거절 | -1 ~ -3 |
| 영토 침공 | -20 ~ -40 |
| 공동 적 격파 | +10 ~ +25 |
| 결혼 동맹 | +15 ~ +30 |
| 조공 납부 | +5 ~ +10 |
| 배신 (동맹 위반) | -30 ~ -50 |
| 난민 수용 | +5 ~ +15 |
| 자원 약탈 | -15 ~ -30 |
| 고대 유물 반환 | +10 ~ +20 |

## 3. 임계값 이벤트 (Threshold Events)

친밀도가 특정 임계값을 넘을 때 자동으로 이벤트가 발생한다.

### 상승 시 이벤트 (Positive Crossing)

| 임계값 | 발생 이벤트 | 조건 |
|--------|-----------|------|
| 0 돌파 (냉전→중립) | 외교관 파견 가능 | 양측 동의 필요 |
| +30 돌파 (중립→우호) | 교역로 개설 | 자동 |
| +60 돌파 (우호→동맹) | 동맹 조약 체결 제안 | 양측 동의 필요 |
| +80 돌파 (동맹→혈맹) | 혈맹 의식 | 특수 이벤트, 희귀 |

### 하락 시 이벤트 (Negative Crossing)

| 임계값 | 발생 이벤트 | 조건 |
|--------|-----------|------|
| -10 돌파 (중립→냉전) | 외교 마찰 발생 | 자동 |
| -30 돌파 (냉전→적대) | 교역 금지령 | 자동 |
| -60 돌파 (적대→전쟁) | 선전포고 | 자동 + 이벤트 로그 |

## 4. 비우호적 → 우호 전환 (특수 시나리오)

**핵심**: 비우호적(-50)이어도 특정 조건에서 우호(+30)로 전환 가능하다.

### 전환 트리거 시나리오

| 시나리오 | 설명 | 친밀도 변화 |
|---------|------|-----------|
| **공동의 적** | 두 종족 모두 제3 세력에게 위협받을 때 | +20 ~ +40 |
| **인질 구출** | 적대 종족의 인질을 구출해줄 때 | +15 ~ +25 |
| **재난 구조** | 대기근/역병으로 위기에 처한 적대 종족을 도울 때 | +20 ~ +35 |
| **왕족 인연** | 지도자 간의 개인적 연결 | +10 ~ +20 |
| **고대 계약** | 오래된 평화 조약 발견 | +15 ~ +30 |
| **문화 교류** | 예술·종교의 공통점 발견 | +5 ~ +15 |

## 5. 종족별 외교 성향 (Diplomacy Trait)

각 종족은 고유한 외교 성향을 가진다.

| 속성 | 설명 | 범위 |
|------|------|------|
| `trust_rate` | 친밀도 상승 속도 배수 | 0.5 ~ 2.0 |
| `grudge_rate` | 친밀도 하락 속도 배수 | 0.5 ~ 2.0 |
| `memory_duration` | 과거 사건 기억 유지 틱 | 100 ~ 1000 |
| `betrayal_penalty` | 배신당했을 때 추가 하락 | 0 ~ 50 |

### 예시

```yaml
# 엘프: 천천히 믿고, 배신은 절대 못 잊는다
diplomacy_trait:
  trust_rate: 0.7      # 신뢰 쌓기 느림
  grudge_rate: 1.5     # 원한 쌓기 빠름
  memory_duration: 800 # 오래 기억함
  betrayal_penalty: 40 # 배신 시 추가 -40

# 인간: 실용적, 빠르게 화해 가능
diplomacy_trait:
  trust_rate: 1.0
  grudge_rate: 1.0
  memory_duration: 200
  betrayal_penalty: 15
```

## 6. 다자 외교 (Multi-Party Diplomacy)

### 동맹 체인
- A↔B 동맹이고 B↔C 동맹이면 A↔C에 +10 보정
- A↔B 동맹이고 A가 C와 전쟁이면 B에게 참전 요청 가능

### 전쟁 전파
- A가 B를 공격 시, B의 동맹 C에게 참전 요청 이벤트 발생
- C의 참전 여부는 A↔C 친밀도와 B↔C 친밀도 비율로 결정

```python
# 참전 확률 계산 예시
join_probability = (bc_affinity - ac_affinity + 100) / 200
# bc +80, ac -50 → (80 - (-50) + 100) / 200 = 1.15 → 확실히 참전
# bc +30, ac -10 → (30 - (-10) + 100) / 200 = 0.7 → 70% 확률 참전
```

## 7. 외교 YAML 설정 예시

```yaml
# configs/diplomacy/initial_relations.yaml
# 초기 종족 간 친밀도 설정 (미설정 시 기본값 0)
relations:
  - from: human
    to: elf
    affinity: 30      # 인간과 엘프는 기본 우호
  - from: human
    to: dwarf
    affinity: 40
  - from: elf
    to: orc
    affinity: -60     # 엘프와 오크는 적대
  - from: dragon
    to: human
    affinity: -20     # 드래곤은 인간을 약간 무시
  - from: undead
    to: holy_angel    # 언데드와 천사는 전쟁
    affinity: -90
```
