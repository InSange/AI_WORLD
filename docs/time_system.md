# WorldAI — 시간 시스템 설계 (Time System)

> 최종 확정: 2026-04-06 | 상태: Confirmed

---

## 1. 핵심 확정 사항

| 항목 | 값 | 근거 |
|------|----|------|
| **1틱 단위** | 1시간 | 플레이어 1행동 = 1시간 게임 내 시간 |
| **1일** | 24틱 | 24시간 = 24틱 |
| **1계절** | 2160틱 | 90일 × 24시간 |
| **1년** | 8640틱 | 4계절 × 2160틱 |
| **온라인 전용** | 예 | 플레이어 행동 시에만 틱 진행. 오프라인 시 시간 정지 |

> `tick_unit`은 world YAML에서 설정 가능하게 구현. 기본값 `hour`.

## 2. 낮/밤 사이클

### 계절별 일조 시간

```python
DAY_HOURS: dict[Season, tuple[int, int]] = {
    Season.SPRING: (6,  18),  # 12시간 낮  │ 06:00 ~ 18:00
    Season.SUMMER: (4,  20),  # 16시간 낮  │ 04:00 ~ 20:00
    Season.AUTUMN: (6,  18),  # 12시간 낮  │ 06:00 ~ 18:00
    Season.WINTER: (8,  16),  #  8시간 낮  │ 08:00 ~ 16:00
}

# 하루 중 시각 계산
hour_of_day = tick % 24  # 0 ~ 23
```

### 시간대 5구간 (DayPhase)

| 구간 | ID | 시각 | 특징 |
|------|----|------|------|
| 깊은 밤 | `DEEP_NIGHT` | 00 ~ 04 | 언데드·악마 최강. 위험 지역 진입 위험 상승 |
| 새벽 | `DAWN` | 04 ~ 07 | 드래곤 사냥 시작. 모험가 출발 |
| 낮 | `DAY` | 07 ~ 17 | 상인·교역·건설 활성. 농업 이벤트 |
| 황혼 | `DUSK` | 17 ~ 20 | 기사단 귀환. 야간 경계 전환 |
| 밤 | `NIGHT` | 20 ~ 24 | 페어리 달빛 마법. 정탐·정보 수집 |

> 계절에 따라 낮 구간이 확장/축소됨. 여름 `DAWN`은 04시부터, 겨울은 08시부터.

## 4. 낮/밤이 영향을 주는 시스템

### 전투
- 언데드의 야간 전투력: DAY 기준 대비 10배 (0.2 → 2.0)
- 인간의 야간 전투: -50% (조명 없음)
- 드래곤: 새벽/황혼에 공격 선호 (사냥 본능)

### 이벤트 발생 확률
- 상인 이동 이벤트: `DAY`에만 발생 (상인 타입 유동 인구)
- 습격 이벤트: `NIGHT`/`DEEP_NIGHT`에 확률 1.5배
- 몬스터 출현: `DEEP_NIGHT`에 확률 2배
- 모험가 탐험: `DAWN`에 출발, `DUSK`에 귀환

### 교역
- 상인은 `DAY` 외에는 이동 불가 (기본값)
- 드워프 상인: 지하 교역로 이용 → 시간대 무관
- 언데드 교역: `NIGHT`/`DEEP_NIGHT`에만 가능

## 5. 유동 인구 타입 (Population Flow Type)

각 파벌(Faction)의 인구는 **고정 인구**와 **유동 인구**로 구성된다.

| 타입 | ID | 비율 기본값 | 이동 확률 | 역할 | 이벤트 |
|------|----|------------|---------|------|--------|
| **정착민** | `SETTLER` | 70% | 매우 낮음 (3%) | 농업·생산·세금 기반 | 기근/재해 시 피난 |
| **상인** | `MERCHANT` | 10% | 높음 (60%) | 마을 간 교역 | 교역 이벤트, 친밀도 상승 |
| **모험가** | `ADVENTURER` | 8% | 중간 (35%) | 던전·탐험 | 초월자 이벤트 후보, 유적 발견 |
| **군인** | `MILITARY` | 10% | 명령 기반 | 전투·영토 방어·점령 | 전쟁 이벤트 |
| **이주민** | `REFUGEE` | 2% | 매우 높음 (90%) | 전쟁·기근으로 이탈 | 파벌 인구 재배치 |

### 이동 메커니즘
```
인구가 늘어도 → 정착민은 마을에 머묾
상인·모험가 → 인구 증가 시 비율 유지하며 인근 파벌로 이동 가능
기근 발생    → 정착민 일부가 이주민(REFUGEE)으로 전환 → 타 파벌로 이동
전쟁 패배   → 군인 감소 + 정착민 일부 이주민 전환
```

### 군집 규모별 군인 비율
```
마을 (village):  5%  — 동네 민병대 수준
읍   (town):    10%  — 영주 직속 소부대
도시 (city):    12%  — 정규군 주둔
왕국 (kingdom): 15%  — 왕국 기사단
제국 (empire):  18%  — 제국 군단
```

## 6. 이벤트 확률 재조정 (1틱=1시간 기준)

기존 이벤트 확률들이 "추상 틱" 기준이라 모두 재조정 필요.

| 이벤트 | 기존 | 재조정 | 이유 |
|--------|------|--------|------|
| 습격 주기 | 15틱 | **72틱 (3일)** | 매일 습격은 너무 잦음 |
| 영역 확장 체크 | 3%/틱 (85% 이상) | **일 1회 체크, 5%** | 시간 단위로 너무 잦음 |
| 기근 발생 | 5%/틱 (95% 이상) | **일 1회 체크, 10%** | 동일 이유 |
| 역병 주기 | 200틱 | **720틱 (30일)** | 8일마다는 너무 잦음 |
| 몬스터 출현 | 100틱마다 15% | **240틱 (10일)마다 20%** | 대략 월 2회 |
| 교단 교세 전파 | 30틱 | **168틱 (7일)** | 주 1회 |

## 7. 플레이어-서버 상호작용 모델

```
[플레이어 접속]
      │
      ├─ 이전 세션 마지막 상태 로드
      │   (틱 번호, 월드 상태, 플레이어 위치 그리드)
      │
      V
[플레이어 행동 선택]
      │  POST /player/action { grid_x, grid_y, action }
      │
      V
[서버: 틱 처리]
      ├─ 플레이어 행동 → 해당 그리드 적용
      ├─ tick_world() → 전체 월드 1시간 진행
      ├─ 낮/밤 phase 체크 → 종족 활성도 적용
      └─ 플레이어 그리드 이벤트 체크
      │
      V
[이벤트 있으면 플레이어에게 선택지 반환]
  - 이벤트 참여 / 무시 / 다른 그리드 이동
```

## 8. YAML 설정 스키마

```yaml
# configs/worlds/asteria.yaml (추가)
time_config:
  tick_unit: "hour"          # 1틱 = 1시간
  hours_per_day: 24
  days_per_season: 90
  # 자동 계산됨:
  # ticks_per_day: 24
  # ticks_per_season: 2160
  # ticks_per_year: 8640

  online_only: true           # 플레이어 접속 시에만 틱 진행
  session_based: true         # 세션 기반 진행
```
