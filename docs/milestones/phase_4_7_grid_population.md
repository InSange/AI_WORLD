## 1. 200x200 그리드 맵 시스템

전 지면을 40,000개의 타일로 관리하며, 위도(Latitude) 기반의 점진적 지형 생성 알고리즘을 도입했습니다.
(초기 설계는 100x80이었으나 Phase 5에서 200x200으로 확장됨)

### 지형 생성 규칙 (Latitude Gradient)
- **북부 (Y: 0~25)**: 설원(Snow), 툰드라(Tundra). 남쪽으로 갈수록 평원이 섞임.
- **중앙 (Y: 25~55)**: 평원(Plains), 숲(Forest). 가장 거주에 적합한 지역.
- **남부 (Y: 55~80)**: 사막(Desert), 황무지(Wasteland), 열대(Tropical). 북쪽으로 갈수록 평원이 섞임.
- **공통**: 산맥(Mountain)과 물(Water)이 확률적으로 무작위 분포.

## 2. 세그먼트 기반 인구 모델

단일 `float` 값으로 관리되던 인구를 5가지 유동 타입으로 분산 관리합니다.

| 타입 | 명칭 | 특성 |
|------|------|------|
| **Settler** | 정착민 | 파벌의 근간. 이동하지 않고 생산에 집중. |
| **Merchant** | 상인 | 파벌 간 이동 빈도 높음. 경제/문화 전파. |
| **Adventurer** | 모험가 | 자유로운 이동. 몬스터 토벌 및 탐험 기여. |
| **Military** | 군인 | 훈련/충원에 의해 변동. 전투 시 실질적 전력. |
| **Refugee** | 피난민 | 재난/전쟁 발생 시 급증. 높은 이동성. |

### 인구 자동 분배 (Distribution)
파벌 생성 시 규모([SettlementScale](file:///c:/Users/선우/source/repos/WorldAI/apps/worldai/src/core/models.py))에 따라 인구 타입 비율이 자동으로 설정됩니다.
- 예: **전초기지(Outpost)**는 군인 비율 40% 이상, **도시(City)**는 상인 및 정착민 중심.

## 3. 유동 인구 이동 (Migration)

매 틱(1시간)마다 '이동 가능' 속성을 가진 인구 세그먼트가 이동을 시도합니다.

- **규칙**: 1틱당 인접 1타일 이동.
- **로직**: 현재 파벌에서 이탈하여 인근 파벌(같은 종족 우선)로 전입하거나, 타일 위를 부유함.
- **목적**: 대시보드에서 실시간으로 인구가 대도시로 몰리거나 전란을 피해 흩어지는 모습을 시각화하기 위함.

## 4. API 엔드포인트 변경 사항

- `GET /world/map`: 전체 맵의 지형 인덱스 리스트(`data`) 반환.
- `GET /world`: `map` 필드 추가 (WorldMapSchema).
- `GET /factions`: 각 파벌 응답에 `population_segments` 상세 리스트 포함.

## 5. 다음 작업자를 위한 가이드

이 시스템은 **Phase 5: 웹 대시보드** 연동을 목표로 설계되었습니다.
- 프론트엔드에서 8,000개 타일을 효율적으로 그리려면 Canvas API 사용을 권장합니다.
- `PopulationSegment`의 `count`는 성장에 의해 `float` 값이 될 수 있으므로, UI 표시 시 `int()` 처리가 필요합니다.
- 이동 로직은 현재 `FactionManager._handle_migration`에 기본 구현되어 있으며, 향후 지형별 이동 속도 차등(산맥 통과 불가 등)을 레이어로 추가할 수 있습니다.
